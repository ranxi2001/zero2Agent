# 框架调研

这个模块是一份**广度优先的框架地图**。

不是把每个框架都学深，而是搞清楚它们各自在解决什么问题、适合什么场景，以及和你已经知道的东西有什么区别。

## 怎么读这个模块

每篇文章的结构一致：

1. **是什么** — 核心定位和解决的问题
2. **怎么装** — 安装和初始化
3. **核心用法** — 最小可运行示例
4. **关键概念** — 这个框架特有的抽象
5. **优缺点** — 适合什么场景，不适合什么

不需要按顺序读，按感兴趣的框架跳着看就行。
