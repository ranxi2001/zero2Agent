本三，社招agent 面经。
成都某电商公司，直接技术负责人面试
1. 自我介绍
2. 了解我们做什么的吗
3. 会用hive、spark吗
4. 开始打压了，面试官开始表达自己对agent 的看法，他认为该怎么样怎么样
5. 问部门组织架构
6. 问项目复杂度，你们测试是谁，为什么不用ai 来测试，面试官表示他们现在都是类似ai 测试
7. 后续就是我急眼了，两个人互相在办公室互相抬杠

成都某智能客服+外呼公司，也是技术负责人面试
1. 自我介绍
2. 为什么离职
3. sse 和 websocket 、单次调用区别
4. Rag 流程描述
5. 你做过什么，巴拉巴拉一大堆
6. 怎么调用openai 的，了解大模型网关没有
7. 面试官表示要看我的在线部署项目，我说我github有，docker 快速run 一下就可以，他死活不动，哈哈哈，一副吊炸天样子

成都某大厂子公司，技术负责人面试
1. 自我介绍
2. 为什么离职
3. 聊聊你的项目
4. 你怎么降低时延，怎么优化rag，prompt 这些。。。 
5. 面试大概半小时，面试官没什么兴趣，反问环节，原来是做内部智能客服面向技术支持的，确实不匹配

总结一下，大部分都是半小时。成都目前很多挂羊头卖狗肉的agent 岗位，实际面试官自己都可能不太清楚agent 开发。我觉得面试应该是充分理解并且互相尊重的过程，但很遗憾，成都的面试官实在难评。 好在自己薪资尚可，出去只是看看市场情况
