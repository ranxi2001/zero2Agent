实验室项目深挖
1.请介绍一下“多模态数据采集与实时处理”项目，这里的“多模态”是指哪部分？“实时处理”是什么意思？
2.大概有多少台指纹机和摄像头？
3.你的定时任务是多久访问一次？
4.你做这个项目遇到的难点是什么？为什么要用 ThreadPoolExecutor？为什么要用 Redis Hash？
5.假设你设置了 5 个线程，刚好有 5 台机器故障，线程池锁死了怎么办？有没有设置超时机制？
6.关于“集群环境下通过 Redis 分布式锁保证任务不重复执行”，你是怎么实现的？有没有用 Lua 脚本写分布式锁？

Text-to-SQL Agent 深度探讨
7.请解释一下你的 Text-to-SQL Agent 项目代码。
8.在开发过程中，你发现哪些机制（如 ReAct、反思）对结果生成有实际提升？
9.你有没有用数据集（Dataset）来测试系统的性能？
10.你目前的 RAG 召回率（Recall）和精确率（Precision）大概是多少？表级别的精确率有看过吗？
11.你有没有对比过做 RAG 之前（把所有表结构塞进上下文）和做 RAG 之后的效果差异？
12.你关注到最近关于“用 Agent 直接搜索文档（如 grep/rg）”来替代向量检索的新思路了吗？

算法与基础
13.请解释一下 HashMap 的底层数据结构。
14.为什么 HashMap 的链表要转成红黑树？
15.为什么 HashMap 不直接扩容，而要选择转成红黑树？
（算法题）场景：物流报价，从上海到美国再到墨西哥，或者直达，价格不同。如何计算出最优价格？（这里我归类为图的最短路径算法，说了DFS/BFS都可以，然后在考虑价格，就是有权图，bfs面试官说不行，我再说 Dijkstra）
AI 观点与结尾
16.你对模型微调（Fine-tuning）怎么看？（这里是面试官向我介绍他们的项目的时候我提出在他们的场景下可不可以用微调，面试官说可以，并且他们也有尝试过，但一次微调之后只能保持三个月领先）
