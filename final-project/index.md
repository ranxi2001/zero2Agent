# final-project

最终实战会围绕一个真实业务问题展开：

**加密货币市场风控 Agent 开发**

## 为什么选这个方向

这个场景同时具备下面几个特征：

- 数据源复杂
- 事件驱动强
- 风险定义不能只靠一句 Prompt
- 输出必须可解释、可审计、可人工接管

这使它很适合作为 Agent 工程实践题，而不是只做一个能聊天的助手。

## 计划拆分的能力模块

- 市场信息采集
- 事件检测与结构化
- 风险信号提取
- 多源分析工作流
- 告警生成与解释
- 人工审核与边界控制
